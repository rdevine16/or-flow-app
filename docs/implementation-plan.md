> Generated by planning interview on 2026-03-01

## Summary

Build an HL7v2 SIU message listener for Epic OpTime surgical scheduling data. Replaces the FHIR-based approach (patient-centric, doesn't expose surgical scheduling). **Primary goal:** test harness to validate Epic HL7v2 field mappings without paying $1,900/year for Epic sandbox. 7 phases, ~10-12 sessions.

## Phase 1: HL7v2 Parser & Message Models

**Complexity:** Medium

### What it does
- Generic HL7v2 message parser that decodes pipe-delimited messages into typed TypeScript objects
- Handles all 5 delimiters: field `|`, component `^`, subcomponent `&`, repetition `~`, escape `\`
- SIU-specific parser that extracts surgical case data and routes trigger events (S12-S16)
- ACK (acknowledgment) response generator (AA/AE/AR)
- Validates required fields, handles optional/missing segments gracefully

### Files touched

| File | Change |
|------|--------|
| `lib/hl7v2/types.ts` | **Create** ŌĆö TypeScript interfaces: `MSHSegment`, `SCHSegment`, `PIDSegment`, `PV1Segment`, `DG1Segment`, `RGSSegment`, `AISSegment`, `AIGSegment`, `AILSegment`, `AIPSegment`, `SIUMessage`, `SIUTriggerEvent`, `HL7v2ParseResult` |
| `lib/hl7v2/parser.ts` | **Create** ŌĆö Generic parser: `parseMessage()`, `parseSegment()`, `parseField()`, `parseComponent()`, `parseDateTime()` |
| `lib/hl7v2/siu-parser.ts` | **Create** ŌĆö SIU-specific: `parseSIUMessage()`, trigger event routing, required field validation |
| `lib/hl7v2/ack-generator.ts` | **Create** ŌĆö `generateACK()` producing AA/AE/AR with correct MSH control IDs and timestamps |
| `lib/hl7v2/__tests__/parser.test.ts` | **Create** ŌĆö Generic parser unit tests |
| `lib/hl7v2/__tests__/siu-parser.test.ts` | **Create** ŌĆö SIU parser tests for all trigger events |
| `lib/hl7v2/__tests__/ack-generator.test.ts` | **Create** ŌĆö ACK generator tests |

### Key implementation notes
- Parse MSH-9 to identify message type (`SIU^S12`, etc.) and MSH-12 for HL7 version
- SCH-11 contains timing as `^^duration^startDT^endDT` ŌĆö parse carefully
- PID-5 is `LAST^FIRST^MIDDLE^^` ŌĆö split by component separator
- AIP segments repeat (one per personnel role) ŌĆö collect into array
- DG1 segments repeat (multiple diagnoses) ŌĆö collect into array
- All date/time parsing: `YYYYMMDDHHMMSS` ŌåÆ ISO 8601
- Empty fields between pipes (`||`) = null, not empty string

### Commit message
`feat(hl7v2): phase 1 - HL7v2 message parser and SIU type definitions`

### 3-stage test gate
1. **Unit:** Parse valid SIU^S12 message, verify all fields. Parse messages with missing optional segments (no DG1, no AIG). Parse messages with repeating AIP segments. Handle malformed messages (return error, don't crash). ACK generation with correct control ID.
2. **Integration:** Round-trip: build message string ŌåÆ parse ŌåÆ verify structured output matches input. Test all 5 trigger events (S12, S13, S14, S15, S16).
3. **Workflow:** Parse the example Epic OpTime SIU^S12 from the plan doc ŌåÆ extract all field values ŌåÆ verify they map correctly to ORbit column values per the field mapping table.

## Phase 3: Case Import Service

**Complexity:** Heavy (2 sessions)

### What it does
- Maps parsed SIU messages to ORbit database operations (create/update/cancel cases)
- Patient matching: MRN ŌåÆ name+DOB ŌåÆ create new
- Surgeon matching: NPI ŌåÆ fuzzy name match (reuses `lib/epic/auto-matcher.ts` Levenshtein engine)
- Procedure matching: CPT code ŌåÆ fuzzy name match
- OR room matching: normalized name match ("OR3" = "OR 3" = "Operating Room 3")
- Idempotent: same S12 twice ŌåÆ only one case created
- Review queue flow: unmatched entities ŌåÆ `pending_review` with suggestions

### Files touched

| File | Change |
|------|--------|
| `lib/integrations/epic/case-import-service.ts` | **Create** ŌĆö Main orchestrator: `handleSIUMessage()`, routes by trigger event (S12ŌåÆcreate, S13/S14ŌåÆupdate, S15/S16ŌåÆcancel) |
| `lib/integrations/epic/patient-matcher.ts` | **Create** ŌĆö `matchOrCreatePatient()`: MRN match ŌåÆ name+DOB match ŌåÆ create new |
| `lib/integrations/epic/provider-matcher.ts` | **Create** ŌĆö `matchSurgeon()`: NPI match ŌåÆ Levenshtein name match ŌåÆ suggest/unmatched |
| `lib/integrations/epic/procedure-matcher.ts` | **Create** ŌĆö `matchProcedure()`: CPT match ŌåÆ Levenshtein name match ŌåÆ suggest/unmatched |
| `lib/integrations/epic/room-matcher.ts` | **Create** ŌĆö `matchRoom()`: normalized name match (strip spaces, "OR"/"Operating Room" equivalence) |
| `lib/integrations/shared/integration-logger.ts` | **Create** ŌĆö Writes to `ehr_integration_log`, builds `review_notes` JSONB with entity suggestions |
| `lib/integrations/epic/__tests__/case-import-service.test.ts` | **Create** ŌĆö Full import service tests |
| `lib/integrations/epic/__tests__/patient-matcher.test.ts` | **Create** ŌĆö Patient matching tests |
| `lib/integrations/epic/__tests__/provider-matcher.test.ts` | **Create** ŌĆö Provider matching tests |
| `lib/integrations/epic/__tests__/room-matcher.test.ts` | **Create** ŌĆö Room matching tests |

### Key implementation notes
- **Reuse** `lib/epic/auto-matcher.ts` ŌĆö import `similarityScore` and `findBestMatch` functions
- **Check `ehr_entity_mappings` first** for each entity before fuzzy matching. If mapping exists, auto-resolve.
- S12 flow: parse ŌåÆ check entity mappings ŌåÆ fuzzy match remaining ŌåÆ if ALL matched: create case + patient, status='processed' ŌåÆ if ANY unmatched: status='pending_review' with review_notes
- **Patient demographics:** If MRN matches but name/DOB differ, **flag as `pending_review`** (cautious HIPAA approach). Include `demographics_mismatch` in review_notes.
- S13/S14 (update): look up by `external_case_id` + `facility_id`. If not found, treat as S12.
- S15/S16 (cancel): look up by `external_case_id` + `facility_id`. If not found, log as 'ignored'.
- **Dedup (belt-and-suspenders):** First check `message_control_id` (MSH-10) in `ehr_integration_log` ŌĆö if already processed, return cached ACK immediately. Then check `external_case_id + facility_id` on `cases` table.
- Case number format: `"HL7-{external_case_id}"`
- **Full trigger pipeline:** Use existing `create_case_with_milestones` RPC for case creation ŌĆö triggers milestone pipeline. Imported cases are first-class citizens.
- Do NOT set `data_validated = true` on import ŌĆö that's for post-surgery milestone recording
- Wrap create/update in transaction. All operations write to `ehr_integration_log`.
- Auto-save high-confidence matches (>=0.90) to `ehr_entity_mappings` for future auto-resolution.

### Commit message
`feat(hl7v2): phase 3 - case import service with patient/provider/procedure matching`

### 3-stage test gate
1. **Unit:** Each matcher in isolation ŌĆö exact match, fuzzy match, no match. Room name normalization ("OR3" = "OR 3" = "Operating Room 3"). Idempotency check (duplicate message_control_id rejected).
2. **Integration:** Full pipeline: parsed SIU^S12 ŌåÆ import service ŌåÆ verify case in DB with correct fields. Verify patient matched by MRN. Verify S12ŌåÆS14ŌåÆS15 lifecycle (case goes scheduledŌåÆupdatedŌåÆcancelled).
3. **Workflow:** S12 with unmatched surgeon ŌåÆ verify `pending_review` status with correct `review_notes` JSONB ŌåÆ verify case NOT created yet ŌåÆ simulate entity resolution ŌåÆ approve ŌåÆ verify case created with correct surgeon.

## Phase 4: HL7v2 Listener Edge Function

**Complexity:** Medium

### What it does
- Supabase Edge Function receiving HTTP POST with HL7v2 messages
- **Full processing pipeline**: auth ŌåÆ parse ŌåÆ match entities ŌåÆ create/update case ŌåÆ log ŌåÆ return ACK
- Authenticates via API key (header) or Basic Auth
- Uses **service role key** for DB access, enforces facility scoping in application code
- Logs every message to `ehr_integration_log`
- Same code path for test harness and production ŌĆö zero rework when a customer connects

### Files touched

| File | Change |
|------|--------|
| `supabase/functions/hl7v2-listener/index.ts` | **Create** ŌĆö Main Edge Function handler: Deno.serve, CORS, parse body, auth, full processing pipeline, ACK response |
| `supabase/functions/hl7v2-listener/auth.ts` | **Create** ŌĆö `authenticateRequest()`: API key header ŌåÆ Basic Auth fallback ŌåÆ facility lookup |
| `supabase/functions/hl7v2-listener/parser.ts` | **Create** ŌĆö Duplicated generic HL7v2 parser (Deno-compatible copy from lib/hl7v2/) |
| `supabase/functions/hl7v2-listener/types.ts` | **Create** ŌĆö Duplicated type definitions |
| `supabase/functions/hl7v2-listener/siu-parser.ts` | **Create** ŌĆö Duplicated SIU-specific parser |
| `supabase/functions/hl7v2-listener/ack-generator.ts` | **Create** ŌĆö Duplicated ACK generator |
| `supabase/functions/hl7v2-listener/import-service.ts` | **Create** ŌĆö Duplicated/adapted import service for Deno runtime |
| `supabase/functions/hl7v2-listener/rate-limiter.ts` | **Create** ŌĆö Simple time-window rate limiter |

### Key implementation notes
- **Content-Types accepted**: `application/hl7-v2`, `text/plain`, `x-application/hl7-v2+er7`
- **Auth flow**: Check `X-Integration-Key` header ŌåÆ if missing, check `Authorization: Basic ...` ŌåÆ query `ehr_integrations` by API key ŌåÆ reject if not found ŌåÆ facility determined from match
- **DB access**: `SUPABASE_SERVICE_ROLE_KEY` as Edge Function secret. Bypasses RLS. Facility scoping enforced in application code.
- **Response**: Always return HL7v2 ACK in body (AA for success, AE for processing error, AR for auth failure). Content-Type: `application/hl7-v2`
- **Rate limiting**: Max 100 messages/minute per facility (configurable in `config` JSONB)
- **No JWT auth** ŌĆö set `verify_jwt: false` since this uses API key auth (Epic engines don't have Supabase accounts)
- **Code duplication**: Parser, types, SIU parser, ACK generator, and import service duplicated into Edge Function directory. Deno runtime can't import from lib/. Keep both copies in sync manually.
- Error responses: malformed message ŌåÆ AE ACK + log with 'error' status + descriptive error_message

### Commit message
`feat(hl7v2): phase 4 - HL7v2 listener endpoint for receiving Epic messages`

### 3-stage test gate
1. **Unit:** Auth verification with valid/invalid API key. Auth with Basic Auth. Rate limiter counting and rejection. ACK response format for each scenario.
2. **Integration:** POST valid SIU^S12 to Edge Function ŌåÆ verify case created in DB + AA ACK returned. POST with bad API key ŌåÆ verify 401 + AR ACK. POST malformed message ŌåÆ verify AE ACK + error logged.
3. **Workflow:** Send sequence: S12 (create) ŌåÆ S14 (modify surgeon) ŌåÆ S15 (cancel) ŌåÆ verify final case state is cancelled ŌåÆ verify `ehr_integration_log` has 3 entries with correct statuses ŌåÆ verify `last_message_at` updated on `ehr_integrations`.

## Risk Assessment

| Risk | Mitigation |
|------|-----------|
| Epic field positions vary by facility | `ehr_integrations.config` JSONB supports field mapping overrides per facility |
| PHI in raw HL7v2 messages stored in DB | RLS + encryption at rest. Configurable retention policy with automated pg_cron purge (Phase 7). PHI access tracking. |
| 8 triggers on `cases` table may interfere with import | Use existing `create_case_with_milestones` RPC. Test trigger cascade with imported cases. Full trigger pipeline confirmed as desired. |
| Fuzzy matching creates false positives | Confidence threshold (0.90+ auto-map to ehr_entity_mappings, 0.70-0.89 suggest). All suggestions shown in review queue for human verification. |
| Edge Function cold start latency | Epic integration engines have retry logic. ACK timeout 30-60s. Supabase Edge Functions start <1s. |
| Rate limiting on Supabase Edge Functions | Typical ASC sends 10-30 messages/day. Well within platform limits. Per-facility rate limiting configurable. |
| Code duplication (lib/ Ōåö Edge Function) | Parser, types, SIU parser, ACK generator duplicated into Edge Function. Keep both in sync manually. Consider shared package when monorepo tooling is adopted. |
| Patient demographics mismatch | Cautious approach: flag for review. Admin verifies before case is created. Prevents wrong patient association. |
| Message retry storms (integration engine retries) | Belt-and-suspenders dedup: message_control_id (MSH-10) + external_case_id. Cached ACK returned for duplicates. |
