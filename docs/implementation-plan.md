# Implementation Plan: Epic HL7v2 Surgical Case Scheduling Integration

> Generated by planning interview on 2026-03-01
> Updated by /review deep design review on 2026-03-01

## Summary

Build an HL7v2 SIU message listener for Epic OpTime surgical scheduling data. Replaces the FHIR-based approach (patient-centric, doesn't expose surgical scheduling). **Primary goal:** test harness to validate Epic HL7v2 field mappings without paying $1,900/year for Epic sandbox. Phases 1-7 build the core pipeline. Phases 8-11 add a database-driven Test Data Manager in global admin, replacing hardcoded test data with CRUD-able entity pools and schedule entries so you can hand-craft test cases and validate the import pipeline under different facility mapping states (none, partial, complete). 11 phases, ~14-17 sessions.

## Phase 1: HL7v2 Parser & Message Models

**Complexity:** Medium

### What it does
- Generic HL7v2 message parser that decodes pipe-delimited messages into typed TypeScript objects
- Handles all 5 delimiters: field `|`, component `^`, subcomponent `&`, repetition `~`, escape `\`
- SIU-specific parser that extracts surgical case data and routes trigger events (S12-S16)
- ACK (acknowledgment) response generator (AA/AE/AR)
- Validates required fields, handles optional/missing segments gracefully

### Files touched

| File | Change |
|------|--------|
| `lib/hl7v2/types.ts` | **Create** — TypeScript interfaces: `MSHSegment`, `SCHSegment`, `PIDSegment`, `PV1Segment`, `DG1Segment`, `RGSSegment`, `AISSegment`, `AIGSegment`, `AILSegment`, `AIPSegment`, `SIUMessage`, `SIUTriggerEvent`, `HL7v2ParseResult` |
| `lib/hl7v2/parser.ts` | **Create** — Generic parser: `parseMessage()`, `parseSegment()`, `parseField()`, `parseComponent()`, `parseDateTime()` |
| `lib/hl7v2/siu-parser.ts` | **Create** — SIU-specific: `parseSIUMessage()`, trigger event routing, required field validation |
| `lib/hl7v2/ack-generator.ts` | **Create** — `generateACK()` producing AA/AE/AR with correct MSH control IDs and timestamps |
| `lib/hl7v2/__tests__/parser.test.ts` | **Create** — Generic parser unit tests |
| `lib/hl7v2/__tests__/siu-parser.test.ts` | **Create** — SIU parser tests for all trigger events |
| `lib/hl7v2/__tests__/ack-generator.test.ts` | **Create** — ACK generator tests |

### Key implementation notes
- Parse MSH-9 to identify message type (`SIU^S12`, etc.) and MSH-12 for HL7 version
- SCH-11 contains timing as `^^duration^startDT^endDT` — parse carefully
- PID-5 is `LAST^FIRST^MIDDLE^^` — split by component separator
- AIP segments repeat (one per personnel role) — collect into array
- DG1 segments repeat (multiple diagnoses) — collect into array
- All date/time parsing: `YYYYMMDDHHMMSS` → ISO 8601
- Empty fields between pipes (`||`) = null, not empty string

### Commit message
`feat(hl7v2): phase 1 - HL7v2 message parser and SIU type definitions`

### 3-stage test gate
1. **Unit:** Parse valid SIU^S12 message, verify all fields. Parse messages with missing optional segments (no DG1, no AIG). Parse messages with repeating AIP segments. Handle malformed messages (return error, don't crash). ACK generation with correct control ID.
2. **Integration:** Round-trip: build message string → parse → verify structured output matches input. Test all 5 trigger events (S12, S13, S14, S15, S16).
3. **Workflow:** Parse the example Epic OpTime SIU^S12 from the plan doc → extract all field values → verify they map correctly to ORbit column values per the field mapping table.

## Phase 3: Case Import Service

**Complexity:** Heavy (2 sessions)

### What it does
- Maps parsed SIU messages to ORbit database operations (create/update/cancel cases)
- Patient matching: MRN → name+DOB → create new
- Surgeon matching: NPI → fuzzy name match (reuses `lib/epic/auto-matcher.ts` Levenshtein engine)
- Procedure matching: CPT code → fuzzy name match
- OR room matching: normalized name match ("OR3" = "OR 3" = "Operating Room 3")
- Idempotent: same S12 twice → only one case created
- Review queue flow: unmatched entities → `pending_review` with suggestions

### Files touched

| File | Change |
|------|--------|
| `lib/integrations/epic/case-import-service.ts` | **Create** — Main orchestrator: `handleSIUMessage()`, routes by trigger event (S12→create, S13/S14→update, S15/S16→cancel) |
| `lib/integrations/epic/patient-matcher.ts` | **Create** — `matchOrCreatePatient()`: MRN match → name+DOB match → create new |
| `lib/integrations/epic/provider-matcher.ts` | **Create** — `matchSurgeon()`: NPI match → Levenshtein name match → suggest/unmatched |
| `lib/integrations/epic/procedure-matcher.ts` | **Create** — `matchProcedure()`: CPT match → Levenshtein name match → suggest/unmatched |
| `lib/integrations/epic/room-matcher.ts` | **Create** — `matchRoom()`: normalized name match (strip spaces, "OR"/"Operating Room" equivalence) |
| `lib/integrations/shared/integration-logger.ts` | **Create** — Writes to `ehr_integration_log`, builds `review_notes` JSONB with entity suggestions |
| `lib/integrations/epic/__tests__/case-import-service.test.ts` | **Create** — Full import service tests |
| `lib/integrations/epic/__tests__/patient-matcher.test.ts` | **Create** — Patient matching tests |
| `lib/integrations/epic/__tests__/provider-matcher.test.ts` | **Create** — Provider matching tests |
| `lib/integrations/epic/__tests__/room-matcher.test.ts` | **Create** — Room matching tests |

### Key implementation notes
- **Reuse** `lib/epic/auto-matcher.ts` — import `similarityScore` and `findBestMatch` functions
- **Check `ehr_entity_mappings` first** for each entity before fuzzy matching. If mapping exists, auto-resolve.
- S12 flow: parse → check entity mappings → fuzzy match remaining → if ALL matched: create case + patient, status='processed' → if ANY unmatched: status='pending_review' with review_notes
- **Patient demographics:** If MRN matches but name/DOB differ, **flag as `pending_review`** (cautious HIPAA approach). Include `demographics_mismatch` in review_notes.
- S13/S14 (update): look up by `external_case_id` + `facility_id`. If not found, treat as S12.
- S15/S16 (cancel): look up by `external_case_id` + `facility_id`. If not found, log as 'ignored'.
- **Dedup (belt-and-suspenders):** First check `message_control_id` (MSH-10) in `ehr_integration_log` — if already processed, return cached ACK immediately. Then check `external_case_id + facility_id` on `cases` table.
- Case number format: `"HL7-{external_case_id}"`
- **Full trigger pipeline:** Use existing `create_case_with_milestones` RPC for case creation — triggers milestone pipeline. Imported cases are first-class citizens.
- Do NOT set `data_validated = true` on import — that's for post-surgery milestone recording
- Wrap create/update in transaction. All operations write to `ehr_integration_log`.
- Auto-save high-confidence matches (>=0.90) to `ehr_entity_mappings` for future auto-resolution.

### Commit message
`feat(hl7v2): phase 3 - case import service with patient/provider/procedure matching`

### 3-stage test gate
1. **Unit:** Each matcher in isolation — exact match, fuzzy match, no match. Room name normalization ("OR3" = "OR 3" = "Operating Room 3"). Idempotency check (duplicate message_control_id rejected).
2. **Integration:** Full pipeline: parsed SIU^S12 → import service → verify case in DB with correct fields. Verify patient matched by MRN. Verify S12→S14→S15 lifecycle (case goes scheduled→updated→cancelled).
3. **Workflow:** S12 with unmatched surgeon → verify `pending_review` status with correct `review_notes` JSONB → verify case NOT created yet → simulate entity resolution → approve → verify case created with correct surgeon.

## Phase 4: HL7v2 Listener Edge Function

**Complexity:** Medium

### What it does
- Supabase Edge Function receiving HTTP POST with HL7v2 messages
- **Full processing pipeline**: auth → parse → match entities → create/update case → log → return ACK
- Authenticates via API key (header) or Basic Auth
- Uses **service role key** for DB access, enforces facility scoping in application code
- Logs every message to `ehr_integration_log`
- Same code path for test harness and production — zero rework when a customer connects

### Files touched

| File | Change |
|------|--------|
| `supabase/functions/hl7v2-listener/index.ts` | **Create** — Main Edge Function handler: Deno.serve, CORS, parse body, auth, full processing pipeline, ACK response |
| `supabase/functions/hl7v2-listener/auth.ts` | **Create** — `authenticateRequest()`: API key header → Basic Auth fallback → facility lookup |
| `supabase/functions/hl7v2-listener/parser.ts` | **Create** — Duplicated generic HL7v2 parser (Deno-compatible copy from lib/hl7v2/) |
| `supabase/functions/hl7v2-listener/types.ts` | **Create** — Duplicated type definitions |
| `supabase/functions/hl7v2-listener/siu-parser.ts` | **Create** — Duplicated SIU-specific parser |
| `supabase/functions/hl7v2-listener/ack-generator.ts` | **Create** — Duplicated ACK generator |
| `supabase/functions/hl7v2-listener/import-service.ts` | **Create** — Duplicated/adapted import service for Deno runtime |
| `supabase/functions/hl7v2-listener/rate-limiter.ts` | **Create** — Simple time-window rate limiter |

### Key implementation notes
- **Content-Types accepted**: `application/hl7-v2`, `text/plain`, `x-application/hl7-v2+er7`
- **Auth flow**: Check `X-Integration-Key` header → if missing, check `Authorization: Basic ...` → query `ehr_integrations` by API key → reject if not found → facility determined from match
- **DB access**: `SUPABASE_SERVICE_ROLE_KEY` as Edge Function secret. Bypasses RLS. Facility scoping enforced in application code.
- **Response**: Always return HL7v2 ACK in body (AA for success, AE for processing error, AR for auth failure). Content-Type: `application/hl7-v2`
- **Rate limiting**: Max 100 messages/minute per facility (configurable in `config` JSONB)
- **No JWT auth** — set `verify_jwt: false` since this uses API key auth (Epic engines don't have Supabase accounts)
- **Code duplication**: Parser, types, SIU parser, ACK generator, and import service duplicated into Edge Function directory. Deno runtime can't import from lib/. Keep both copies in sync manually.
- Error responses: malformed message → AE ACK + log with 'error' status + descriptive error_message

### Commit message
`feat(hl7v2): phase 4 - HL7v2 listener endpoint for receiving Epic messages`

### 3-stage test gate
1. **Unit:** Auth verification with valid/invalid API key. Auth with Basic Auth. Rate limiter counting and rejection. ACK response format for each scenario.
2. **Integration:** POST valid SIU^S12 to Edge Function → verify case created in DB + AA ACK returned. POST with bad API key → verify 401 + AR ACK. POST malformed message → verify AE ACK + error logged.
3. **Workflow:** Send sequence: S12 (create) → S14 (modify surgeon) → S15 (cancel) → verify final case state is cancelled → verify `ehr_integration_log` has 3 entries with correct statuses → verify `last_message_at` updated on `ehr_integrations`.

```sql
-- Facility-scoped test data pools (each facility gets its own test data)

CREATE TABLE ehr_test_surgeons (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),

  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()

CREATE TABLE ehr_test_procedures (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  facility_id UUID NOT NULL REFERENCES facilities(id) ON DELETE CASCADE,
  name TEXT NOT NULL,                -- "Total knee arthroplasty"
  cpt_code TEXT,                     -- "27447"
  typical_duration_min INTEGER,      -- 120
  specialty TEXT,                    -- "Orthopedics" (for filtering)
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE ehr_test_rooms (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  facility_id UUID NOT NULL REFERENCES facilities(id) ON DELETE CASCADE,
  name TEXT NOT NULL,                -- "OR-3"
  location_code TEXT,                -- "OR3" (Epic AIL segment value)
  room_type TEXT,                    -- "operating_room", "endo_suite", "cath_lab"
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE ehr_test_patients (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  facility_id UUID NOT NULL REFERENCES facilities(id) ON DELETE CASCADE,
  first_name TEXT NOT NULL,
  last_name TEXT NOT NULL,
  mrn TEXT,                          -- "MRN12345"
  date_of_birth DATE,
  gender TEXT CHECK (gender IN ('M', 'F', 'O', 'U')),
  address_line TEXT,                 -- "123 Main St"
  city TEXT,
  state TEXT,
  zip TEXT,
  phone TEXT,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE ehr_test_diagnoses (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  facility_id UUID NOT NULL REFERENCES facilities(id) ON DELETE CASCADE,
  icd10_code TEXT NOT NULL,          -- "M17.11"
  description TEXT NOT NULL,         -- "Primary osteoarthritis, right knee"
  specialty TEXT,                    -- "Orthopedics" (for filtering/grouping)
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE ehr_test_schedules (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  facility_id UUID NOT NULL REFERENCES facilities(id) ON DELETE CASCADE,
  patient_id UUID NOT NULL REFERENCES ehr_test_patients(id) ON DELETE CASCADE,
  surgeon_id UUID NOT NULL REFERENCES ehr_test_surgeons(id) ON DELETE CASCADE,
  procedure_id UUID NOT NULL REFERENCES ehr_test_procedures(id) ON DELETE CASCADE,
  room_id UUID NOT NULL REFERENCES ehr_test_rooms(id) ON DELETE CASCADE,
  diagnosis_id UUID REFERENCES ehr_test_diagnoses(id) ON DELETE SET NULL,
  scheduled_date DATE NOT NULL,
  start_time TIME NOT NULL,          -- "07:30:00"
  duration_min INTEGER NOT NULL,     -- 120
  trigger_event TEXT NOT NULL DEFAULT 'S12'
    CHECK (trigger_event IN ('S12', 'S13', 'S14', 'S15', 'S16')),
  external_case_id TEXT,             -- Auto-generated if null, reuse for S13/S14/S15
  references_schedule_id UUID REFERENCES ehr_test_schedules(id) ON DELETE SET NULL,
    -- For S13/S14/S15: points to the original S12 entry (inherits external_case_id)
  notes TEXT,                        -- Optional description of what this test case tests
  sequence_order INTEGER DEFAULT 0,  -- Controls send order in test harness
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
```

### Key implementation notes
- **Facility-scoped**: All test data tables have `facility_id` FK — each facility gets its own test data pool for testing facility-specific entity mapping scenarios
- **RLS**: Only global admins can CRUD. Facility-scoped queries filter by `facility_id` in DAL functions.
- **Separate diagnoses pool**: `ehr_test_diagnoses` is a standalone table (not embedded in procedures). Schedule entries reference a diagnosis via `diagnosis_id` FK (nullable — diagnosis is optional).
- `ehr_test_schedules.references_schedule_id` links reschedule/modify/cancel entries back to the original S12. The test harness uses this to carry the same `external_case_id` across the message sequence.
- If `external_case_id` is null on an S12 entry, auto-generate one (e.g., `TEST-{uuid short}`)
- `sequence_order` controls the order messages are sent — lets you ensure S12 fires before its S13/S15
- DAL functions follow existing `lib/dal/` patterns: take `supabase` client, return `{ data, error }` tuple
- Add indexes on `facility_id` for all tables

## Risk Assessment

| Risk | Mitigation |
|------|-----------|
| Epic field positions vary by facility | `ehr_integrations.config` JSONB supports field mapping overrides per facility |
| PHI in raw HL7v2 messages stored in DB | RLS + encryption at rest. Configurable retention policy with automated pg_cron purge (Phase 7). PHI access tracking. |
| 8 triggers on `cases` table may interfere with import | Use existing `create_case_with_milestones` RPC. Test trigger cascade with imported cases. Full trigger pipeline confirmed as desired. |
| Fuzzy matching creates false positives | Confidence threshold (0.90+ auto-map to ehr_entity_mappings, 0.70-0.89 suggest). All suggestions shown in review queue for human verification. |
| Edge Function cold start latency | Epic integration engines have retry logic. ACK timeout 30-60s. Supabase Edge Functions start <1s. |
| Rate limiting on Supabase Edge Functions | Typical ASC sends 10-30 messages/day. Well within platform limits. Per-facility rate limiting configurable. |
| Code duplication (lib/ ↔ Edge Function) | Parser, types, SIU parser, ACK generator duplicated into Edge Function. Keep both in sync manually. Consider shared package when monorepo tooling is adopted. |
| Patient demographics mismatch | Cautious approach: flag for review. Admin verifies before case is created. Prevents wrong patient association. |
| Message retry storms (integration engine retries) | Belt-and-suspenders dedup: message_control_id (MSH-10) + external_case_id. Cached ACK returned for duplicates. |
